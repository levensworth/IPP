Condicional:

La estructura de control de flujo por default en el lenguaje es el condicional. Dicha estructura viene a representar el siguiente diagrama de flujo:

IF:

El condicional m√°s simple es el simple if el cual plantea que un bloque de c√≥digo se ejecuta solo si se cumple una condici√≥n.

El siguiente fragmento muestra el uso b√°sico de dicha estructura:

# Complete If statemente

if condition:
	'''
	Do stuff here ...
	'''

En este ejemplo vemos el uso de un simple bloque if , dicha estructura comienza por el uso de la palabra reservada if seguida de una condici√≥n* que arroje alg√∫n valor de verdad (de tipo bool ) y finalmente el uso de : para simboliza que termina la definici√≥n de la condici√≥n a utilizar. Luego, en la linea siguiente (puede dejarse la cantidad de lineas en blanco que se deseen) debe venir un bloque ** de c√≥digo indentado con 4 espacios desde el nivel de indentaci√≥n que tenga el respectivo if .

IF - ELSE :

Vimos el uso de un simple condicional, donde si la condici√≥n se cumple el bloque de c√≥digo se ejecuta pero en caso contrario el programa sigue de largo. Hay casos donde depende del valor de una condici√≥n debemos tomar una decisi√≥n que si o si modifica el flujo del programa. Estos casos hace uso del if-else statement.

El siguiente fragmento muestra su uso:

# If else
if condition:
	'''
	Do stuff here if condition is True
	'''
else:
	'''
	Do stuff here if condition is False
	'''

'''
The code continues here ...
'''

En este fragmento vemos el uso de la palabra reservada if seguida del predicado condition * y luego el uso de los : para simbolizar que la condici√≥n de decisi√≥n finaliz√≥. El bloque que corresponde al c√≥digo que debe ejecutarse en caso de que dicha condici√≥n se cumpla ser√° el primer bloque indentado que el sistema encuentre. Luego debe utilizarse la palabra reservada else seguida de los : para indicar que a continuaci√≥n se encuentra el bloque de c√≥digo que corresponde al caso que la condici√≥n se falsa. Este segundo bloque debe tener el mismo nivel de indentaci√≥n que el que se utiliz√≥ para especificar el bloque de la condici√≥n positiva.

ELIF:

cuando queremos evaluar multiples condiciones las cuales son mutuamente excluyentes, se recomienda el uso del elif . En este caso veamos un ejemplo a modo de clarificar la idea:

Si tenemos que imprimir la relaci√≥n de un n√∫mero con respecto a un intervalo dado que:

‚Üí Si n es menor que el limite inferior, imprimir "Es inferior"

‚Üí Si n est√° dentro de los l√≠mites, imprimir "Incluido"

‚Üí Si n es superior al l√≠mite superior, imprimir "Es superior"

# elif example:

min_interval = 18
max_interval = 100

if n < min_interval:
	print("Es inferior")
elif n > max_interval:
	print("Es superior")
else:
	print("Incluido")

Iteraci√≥n:

La segunda estructura muy utilizada del lenguaje es la iteraci√≥n sobre alg√∫n tipo de colecci√≥n o condici√≥n. En este caso veremos que Python nos permite el uso de 2 estructuras para este objetivo.

FOR:

El primer elemento que veremos es el for dicha estructura tiene como objetivo iterar sobre una colecci√≥n limitada de elementos sobre los que se quiere aplicar alguna transformaci√≥n o se los quiere utilizar para operar.

La estructura es la siguiente:

# For
for loop_item in collection:
	'''
	Code which executes one for each element in the collection.
	take into account that the value of loop_item will change in each iteration to a different element.
	'''

En este caso vemos que la estructura comienza con el uso de la palabra reservada for seguido de la definici√≥n de una variable de iteraci√≥n*, luego se indica la palabra reservada in para indicar el conjunto de iteraci√≥n (collection) y finalmente el uso de : para indicar que termin√≥ la definici√≥n del conjunto.

El bloque de c√≥digo a ejecutar por cada iteraci√≥n debe estar indentado 4 espacio desde el nivel de indentaci√≥n de la palabra for . Es importante remarcar que dicho c√≥digo se va a ejecutar una vez por cada elemento del conjunto collection, en cada ciclo la variable loop_item tomar√° un valor nuevo del conjunto.

While:

La otra estructura que Python nos deja disponibles para iterar es el while dicha estructura tiene uso cuando nos interesa ejecutar un ciclo seg√∫n el valor de una condici√≥n que conocemos o para ciclos infinitos.

Su estructura es la siguiente:

# While
while condition: 
	'''
	Do stuff here
	'''

La estructura comienza con la palabra reservada while seguida de un predicado que se eval√∫e a un valor bool y finalmente el uso de los : para indicar el final de la condici√≥n.

Es importante entender que el predicado se eval√∫a antes de cada iteraci√≥n. Es decir que solo se efect√∫a el ciclo si la condici√≥n es verdadera, en caso contrario omite todo el bloque de c√≥digo indentado 4 espacios desde el nivel de indentaci√≥n que tenga la palabra while .

Funciones [Parte I]:

En todo lenguaje de programaci√≥n el concepto de funciones es primordial y permite acelerar y expandir el desarrollo de software a los niveles que est√°n acostumbrados hoy en d√≠a.

su estructura es la siguiente:

# FUNCTIONS:

# example with arguemnts

	'''
	Do stuff here
	'''


# example with return statement

	'''
	This function, expects name to be a string.

	print("hello World!")

En estos ejemplos vemos que la definici√≥n de una funci√≥n comienza con el uso de la palabra reservada def seguido de un identificador v√°lido que ser√° la forma en que podemos llamar a la funci√≥n, luego vienen los ( y ) entre los cuales pondremos los argumentos de la funci√≥n y finalmente terminamos la estructura con el uso de : para indicar el comienzo del bloque de c√≥digo que ejecuta al funci√≥n.

Esencialmente las funciones son una forma de reutilizar un procedimiento multiples veces sin necesidad de reescribirlo. Esto est√° fuertemente ligado al concepto de Modularizaci√≥n que significa romper un problema complejo es peque√±as partes m√°s simples que son f√°cilmente resolubles y as√≠ obtener una soluci√≥n al problema principal.

Las funciones, como ya dijimos son una forma de condensar un bloque de c√≥digo que queremos reutilizar en multiples ocaciones sin tener que reescribirlo. Las mismas tienen la particularidad de que a veces queremos o necesitamos que el procedimiento que vamos a aplicar sea dependiente de un dato que esta sujeto a cuando aplicamos dicho procedimeinto. Para expresa estas partes m√≥viles del procedimiento que representa una funci√≥n, utilizamos los Argumentos, que simplemente son los identificadores que mencionamos entre los () en al definici√≥n de la funci√≥n. Estos identificadores son variables que se pueden utilizar dentro de la funci√≥n tienen la particularidad de que van a tener el valor que nos indiquen cuando se invoque la funci√≥n.

Invocaci√≥n:

vimos que las funciones son una forma de empaquetar un c√≥digo que utilizaremos muchas veces. Pero como todo paquete, debe tener una forma de recibir mensajes desde afuera y una forma de devolver los resultados que ejerce.

La primera parte, recibir se logra mediante el uso de los Argumentos. Si pensamos en las funciones definidas en el bloque anterior:

# invocation

# which after execution of the function will be

# ======================================
# the above code could be translate into:

En este ejemplo la estamos llamando a ejecutar el bloque de c√≥digo definido bajo el identificador function_return , especificando que el valor de su argumento name en este caso ser√° el string "emily" .

Ahora, dicha funci√≥n ejecuta una operaci√≥n (en este caso es pasar todas las letras a may√∫scula) y nos interesa obtener ese resultado nuevamente. Para ello se hace uso de la palabra reservada return , esta indica que la funci√≥n termina y que devuelve el valor indicado a continuaci√≥n de return como resultado de su invocaci√≥n.

Las funciones pueden tener multiples argumentos, los mismos en esos casos la invocaci√≥n ser√° de la siguiente forma:

En este caso, al invocar la funci√≥n multiple_args tendr√° definido value_1 como valor del identificador arg1 , value_2 como valor del identificador arg2 y value_3 como valor del identificador arg3. Por esto es importante saber el orden de los par√°metros de una funci√≥n.
